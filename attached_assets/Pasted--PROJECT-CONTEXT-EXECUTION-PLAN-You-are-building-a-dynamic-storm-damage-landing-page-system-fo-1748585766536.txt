🔧 PROJECT CONTEXT & EXECUTION PLAN:

You are building a dynamic storm damage landing page system for a roofing company. This project must:

1. Pull trending search phrases from Google Trends daily
2. Filter those trends into two categories: "hail" and "tornado"
3. Cross-check hail-related phrases against NOAA storm reports to confirm:
   - Event is hail
   - Hail size ≥ 2.0"
   - Location is in the Oklahoma City metro area (clustered ZIPs/FIPS)
   - Report occurred within the last 12 months (priority to 1 week)
4. Cross-check tornado phrases against NOAA storm reports to confirm:
   - It was an actual tornado
   - Occurred within the last 14 days
   - Located in the OKC metro
5. Only render a **tornado landing page** if those criteria are met.
6. Always render a **hail landing page** if matching phrase is found.
7. Do NOT use the NOAA API directly — instead, download and parse daily or weekly CSVs from [https://www.ncei.noaa.gov/pub/data/swdi/stormevents/](https://www.ncei.noaa.gov/pub/data/swdi/stormevents/)
8. Page content should update dynamically based on matched trend + event.

---

🧠 SMART LOGIC EXPECTATIONS:

- Extract search intent from trending Google phrases (e.g., "roof damage in Norman")
- Match to verified storm data to determine `storm_type`, `date_of_loss`, `hail_size`, `affected_city`
- Use fallback defaults only if NOAA data is unavailable
- Incorporate phrase value into URL so that pages are pre-rendered or loaded via query param
