🧠 PROJECT CONTEXT:
We’re building a dynamic landing page system for a roofing business in Oklahoma that reacts to recent storm events (hail, wind, tornado) using live data. The goal is to automatically show relevant messaging when a storm has impacted a service area — increasing conversions from visitors who may have been affected.

🔧 OBJECTIVE:
Create a system that pulls real-world storm data (e.g., from NOAA or MRMS) and uses that information to dynamically update landing page content, drawing more traffic and helping homeowners take action at the right time.

🧭 GUIDANCE FOR YOU, REPLIT AI:
The following is a general suggestion — not a rigid plan. You are encouraged to improve or alter this workflow if you see a more elegant, efficient, or scalable way to accomplish the same goal. Our main focus is:  
✅ Timely, relevant storm content  
✅ High-converting lead form  
✅ Automatic updates without manual content editing

🔄 CORE LOGIC:
1. Query a storm event API (e.g., NOAA, MRMS, Hail Reports) for Oklahoma metro regions.
2. Detect most recent impactful event and set these variables:
   - {{DOL}} → Date of Loss (e.g., May 28, 2025)
   - {{X}} → Affected City (e.g., Edmond)
   - {{S}} → Storm type (e.g., “hailstorm” or “tornado”)
   - {{HS}} → Hail size (if applicable, in inches)
   - {{HH}} → Boolean if it’s a hail event
   - {{T}} → Boolean if it’s a wind or tornado event
   - {{HS_less_than_1_5}} → True if hail size is small

3. Inject these values into the landing page dynamically using a templating system (e.g., Handlebars.js, EJS, or your suggested method).
